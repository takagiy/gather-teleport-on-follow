# Chrome Web Store Listing Notes

This document collects the information that must be entered in the Chrome Web Store Developer Dashboard when publishing a new release of this extension. It is meant to be copy-pasted into the dashboard form fields.

The technical package itself is produced with `bun run zip` (see [README.md](../README.md#package-for-the-chrome-web-store)).

## Store listing

### Name

```
Gather Town Teleport on Follow
```

### Summary (≤132 chars)

```
Teleports you next to a player when you click Follow in Gather Town.
```

### Detailed description

```
Gather Town Teleport on Follow extends the Gather web client so that pressing the "フォロー" / Follow button on another player not only follows them but also teleports you next to them instantly.

How it works:
- The extension injects a small script into app.gather.town.
- It subscribes to the client's internal "playerSetsFollowTarget" event.
- When a follow target is set, it reads the target's (map, x, y) and calls the client's teleport API, then clears the follow target after a short delay so you do not keep auto-following.

The extension does not collect, transmit, or store any data. It does not require any Chrome permissions.

Requires Chrome 111 or newer.
```

### Permission justifications

This extension declares **no `permissions`** and **no `host_permissions`** in its manifest. The only thing it does is run a content script on `https://app.gather.town/*` in the page's MAIN world, which does not require a `host_permissions` entry under MV3.

If the dashboard asks about permissions:

- `permissions` justification: _(leave empty — no permissions are declared)_
- `host_permissions` justification: _(leave empty — no host_permissions are declared)_
- Use of remote code: **No.** All code is bundled at build time; no remote scripts are fetched or eval'd.

## Release checklist

1. Bump `version` in `manifest.json` and `package.json` (keep them in sync).
2. Commit and tag (`git tag v<version>`).
3. `bun run check && bun run typecheck` — confirm clean.
4. `bun run zip` — produces `gather-teleport-on-follow-<version>.zip`.
5. Upload the ZIP in the Developer Dashboard → **Package** section.
6. Update store listing fields above if anything changed.
7. Submit for review.
